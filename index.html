Quick example of discovery phase

Device search micro-project

 

Project details:

  • Drupal 8 site

    We need to add a new functionality inside the existing site.

  • External Devise DataBase

    There is API that allow to get information about devices (Specification, Manufactory Location, etc.)

  • The espected result is a page with the list of devices and several search filters

    The page can be accessible for some preveleged user roles or by subscription.

  • Restricted budget for discovery and estimate

    They want some working prototype inside about 10-20h or at least detailed estimate.

 

What should we focused on during the discovery?

 

 

We definitely have the ideas how to organize view with facet filters, how to create subscription with contributed modules.

 

There is only one thing that out of our experience. It's the external DB API.

The result of the discovery phase:

  • A very simple prototype (on the figure)

    A simple text search that return the results form external source.

  • Architectural recommendations

    A set of contributed modules that cover significant part of required functionality, cache architecture and strategy, etc.

  • Detailed estimate and development plan

    List of requred resources, tasks splited by sprints, etc.

Estimation approaches:

    Events functionality, blog functionality, admin area, etc.
